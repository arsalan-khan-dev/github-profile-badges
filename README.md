## 📌 About This Guide

GitHub **Achievements** (also called profile badges) were introduced following the launch of the [Ingenuity Mars Helicopter](https://en.wikipedia.org/wiki/Ingenuity_(helicopter)) in 2021. They display on your public GitHub profile and reflect meaningful contributions and activities across the platform.

**Key facts before you start:**

- There are currently **11 badges** on GitHub, several with multiple progression tiers (Bronze → Silver → Gold).
- By default, badges only track activity in **public repositories**. To include private activity, you must opt in via your profile settings.
- Some badges were tied to one-time historical events and **can no longer be earned**.

## 📖 How to Earn Each Badge

### 🟢 Quickdraw — *Very Easy*

**Requirement:** Close a Pull Request or Issue within 5 minutes of opening it.

```
Step 1 — Go to any repository you own (or create a new test repository)
Step 2 — Click "Issues" → "New Issue"
Step 3 — Enter any title and submit
Step 4 — Immediately click "Close issue" (within 5 minutes)
```

> You can do this entirely solo — no collaborators required. Closing a Pull Request also works if you prefer that route.

---

### 🟢 Public Sponsor — *Very Easy*

**Requirement:** Publicly sponsor any GitHub user, repository, or organization.

```
Step 1 — Find any developer with a "Sponsor" button on their profile
Step 2 — Click "❤️ Sponsor" and select the minimum available tier (often $1/month)
Step 3 — Complete the payment via GitHub's billing system
Step 4 — Badge appears on your profile immediately
```

> You may cancel the sponsorship after the first month — the badge remains on your profile permanently.

---

### 🟡 Pull Shark — *Medium*

**Requirement:** Have 2 pull requests merged into a public repository.

```
Step 1 — Open your repository and create a new branch:
         git checkout -b feature/update-1

Step 2 — Make any change to a file, then push the branch:
         git push origin feature/update-1

Step 3 — Go to GitHub, open a Pull Request from your branch to main

Step 4 — Merge the Pull Request

Step 5 — Repeat steps 1–4 one more time to reach the required 2 merges
```

> Only **merged** PRs count. Closing a PR without merging has no effect toward this badge. You can complete this 100% solo in your own repository.

---

### 🟡 YOLO — *Medium*

**Requirement:** Merge a pull request without requesting or approving a code review.

```
Step 1 — Ensure your repository has no branch protection rules enforcing reviews
Step 2 — Create a branch, make a change, and open a Pull Request
Step 3 — Merge the PR directly, without requesting a review from anyone
```

> If you own the repository and no protection rules are active, you can merge your own PR without needing a second account. The badge is awarded to whoever performs the merge.

---

### 🟡 Galaxy Brain — *Medium*

**Requirement:** Have 2 of your answers marked as accepted in GitHub Discussions on public repositories.

```
Step 1 — Browse GitHub Explore to find public repositories with Discussions enabled
Step 2 — Find open questions you can genuinely answer (focus on topics you know well)
Step 3 — Write a clear, detailed, and helpful response
Step 4 — Wait for the original author to mark your reply as "✅ Answered"
Step 5 — Repeat until you have 2 accepted answers across public repo Discussions
```

> **Important:** Self-accepted answers do not count. The question author (or a maintainer with permission) must mark your reply. The main GitHub Community Discussions forum no longer awards this badge as of February 2024.

---

### 🔴 Pair Extraordinaire — *Hard*

**Requirement:** Be a co-author on a merged pull request.

```
Step 1 — Make a commit and include the following trailer at the end of the commit message:

         Co-authored-by: CollaboratorName <collaborator@email.com>

         (Use your collaborator's exact GitHub display name and email)

Step 2 — Push the branch and open a Pull Request

Step 3 — Get the PR merged
```

> Both the commit author and the listed co-author receive the badge upon merge. The `Co-authored-by:` line must follow the exact format shown above. GitHub Desktop makes it easy to add this through its UI.

---

### 🔴 Starstruck — *Hard*

**Requirement:** Have a repository personally owned by you reach 16 stars.

```
Step 1 — Create a useful, well-documented repository (tools, templates, guides, projects)
Step 2 — Write a clear README with a compelling description, screenshots, and a live demo if applicable
Step 3 — Add relevant topics/tags so it appears in GitHub search results
Step 4 — Share your repository across relevant communities:
         - Reddit: r/github, r/programming, r/webdev, r/opensource
         - Twitter/X using relevant hashtags
         - Dev.to, Hashnode, or your personal blog
         - LinkedIn
         - Developer Discord servers
         - Hacker News (Show HN)
Step 5 — Pin the repository to the top of your GitHub profile
Step 6 — Stay active — respond to issues, accept PRs, keep the README updated
```

> The repository must be **personally owned by you** — repositories under an organization account do not count toward this badge, even if you created them.

## 🙈 Visibility Settings

You have full control over which badges appear on your public profile.

**To hide all achievements at once:**
1. Go to [github.com/settings/profile](https://github.com/settings/profile)
2. Scroll to the **"Achievements"** section
3. Uncheck **"Show Achievements on my profile"**

**To hide a specific badge:**
1. Open the **Achievements** section on your profile
2. Click the badge you want to hide
3. Toggle its visibility off

> Hiding a badge does not remove or delete it. You can re-enable it at any time from the same settings.

## 🔧 Troubleshooting

### Badge Not Appearing After Meeting the Requirements

**Reason 1 — Processing Delay**
Most badges appear instantly, but GitHub can take up to 24–48 hours in some cases. Wait a full day, then check again in a private browsing window to rule out a browser cache issue.

**Reason 2 — Repository Is Private**
Badges only count public repository activity by default. To include private activity:
1. Go to **Settings → Public profile**
2. Enable **"Show private contributions on my profile"**

**Reason 3 — Work Done Inside a Fork**
Activity inside a fork does not count on its own. A PR must be opened against the parent repository and merged there.

**Reason 4 — Organization Repository (Starstruck only)**
For Starstruck, the repository must be personally owned by your account. Stars earned on organization repositories are not counted.

**Reason 5 — PR Was Closed, Not Merged (Pull Shark / YOLO)**
Only pull requests that are **merged** count. A closed-without-merge PR has no effect.

---

### Quick Fix for a Badge Stuck in Sync

If a badge you earned is not displaying correctly:

1. Go to your profile's Achievements tab
2. **Hide** the affected badge
3. Wait a few seconds
4. **Re-enable** it

This forces GitHub to re-sync the badge and resolves most display issues.

## ❓ FAQ

**Do badges expire or disappear over time?**
No. Once earned, all badges are permanent. They never expire. The only exceptions are unobtainable event badges — but if you earned them at the time, they stay.

**Can I earn badges from private repositories?**
Not by default. Enable **"Show private contributions on my profile"** in your settings. This makes your private contribution activity count, while keeping the repository content private.

**How long does it take for a badge to appear?**
Most appear immediately. If it does not show after a page refresh, wait up to 24–48 hours. Check in a private browser tab to rule out cache issues.

**Do organization repository stars count toward Starstruck?**
No. Only stars on repositories directly owned by your personal GitHub account count.

**Can I still earn Galaxy Brain in GitHub Community Discussions?**
No — as of February 2024, GitHub disabled badge awarding there. Use public repository Discussions instead.

**Can I open and close my own issue for Quickdraw?**
Yes. Opening and closing your own issue within 5 minutes is fully valid.

**Do self-accepted answers count for Galaxy Brain?**
No. The answer must be accepted by the person who originally asked the question, or someone with repo-level permission. You cannot accept your own answer.

**Do draft pull requests count toward Pull Shark?**
A draft PR that is fully merged counts. A draft PR that is closed without merging does not.

**If I delete a repository after earning a badge, do I lose the badge?**
No. Badges are tied to your account, not to the repository. Deleting the repo has no effect on earned badges.

**Can both people earn Pair Extraordinaire from a single co-authored commit?**
Yes. Both the commit author and the person listed in `Co-authored-by:` earn the badge when the PR is merged.
